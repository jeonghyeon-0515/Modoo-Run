# Featured Listing 상품안 및 운영 가이드

작성일: 2026-04-16  
기준 서비스: `https://modoo-run.vercel.app`

## 1. 목적

이 문서는 운영팀이 `featured listing` 문의에 답변하고, `/ops/featured` 화면에서 실제 노출을 편성할 때 같은 기준을 쓰도록 정리한 내부 가이드입니다.

핵심 목표는 3가지입니다.

1. 제휴 문의 응대 시 바로 꺼내 쓸 수 있는 상품 설명을 만든다.
2. 운영자가 어떤 대회를 featured로 올릴지 판단 기준을 통일한다.
3. 사용자 화면에서 광고·스폰서·제휴 표기를 헷갈리지 않게 유지한다.

## 2. 현재 제공 가능한 노출면

### A. `/races` 상단 Featured Listing
- 위치: 대회 목록 상단 `주목할 대회` 영역
- 슬롯 수: 메인 1개 + 보조 1개
- 운영 화면: `/ops/featured`
- 노출 요소
  - 라벨(eyebrow)
  - 대회명
  - 일정 / 지역
  - 한 줄 설명(summary)
  - 대회 상세 이동 링크
- 적합한 용도
  - 주최측이 특정 대회를 더 눈에 띄게 보여주고 싶을 때
  - 시즌/지역 큐레이션 안에서 우선 노출이 필요할 때

### B. 대회 상세 정보형 스폰서 슬롯
- 위치: 대회 상세 페이지 정보형 프로모션 슬롯
- 성격: 배너보다 정보성 문구에 가까운 스폰서 슬롯
- 적합한 용도
  - 기록 도구, 준비물, 훈련 보조 장비처럼 러너 맥락과 맞는 협업

### C. 공개 준비물/제휴 가이드
- 위치: `/gear`
- 성격: 공개 가이드 안에서 제휴 링크를 명확히 표시하는 방식
- 적합한 용도
  - 러닝화, 장비, 보조용품처럼 러너가 실제로 찾는 카테고리 연결

## 3. Featured Listing 문의를 받을 때 안내할 내용

### 기본 안내 문구
- 현재 featured listing은 `/races` 상단의 `주목할 대회` 영역에 노출됩니다.
- 운영팀이 메인/보조 2개 슬롯을 직접 편성합니다.
- 광고성 문구보다 러너가 이해하기 쉬운 일정·장소·접수 정보 중심 문구를 우선합니다.
- 가격과 계약 조건은 아직 최종 확정 전이므로 별도 협의가 필요합니다.

### 문의 시 꼭 받아야 할 정보
1. 대회명 / 공식 홈페이지 URL
2. 원하는 노출 유형
   - featured listing
   - 스폰서 슬롯
   - 제휴 가이드 편성
3. 희망 노출 시점 또는 캠페인 일정
4. 강조하고 싶은 핵심 정보
   - 일정
   - 장소
   - 접수 마감
   - 코스 특징
5. 회신 받을 담당자 이름 / 이메일

## 4. `/ops/featured` 편성 기준

운영자가 featured slot을 저장하기 전에 아래를 확인합니다.

### 노출 적합성
- 대회 상세 정보가 실제 공개 상태여야 합니다.
- 일정, 장소, 접수 상태 등 핵심 정보가 비어 있지 않아야 합니다.
- 이미 마감되었거나 사실상 안내 가치가 낮은 대회는 우선순위를 낮춥니다.

### 편성 다양성
- 가능하면 같은 지역/비슷한 성격의 대회만 연속 배치하지 않습니다.
- 메인/보조 슬롯은 비교 탐색에 도움이 되도록 다른 선택지를 보여주는 쪽을 우선합니다.

### 문구 작성 원칙
- 라벨은 사용자가 한눈에 의미를 알 수 있게 씁니다.
  - 예: `Featured Listing`, `지역 추천`, `스폰서`
- 설명 문구는 확인 가능한 사실 위주로 씁니다.
- 과장 표현, 최상급 표현, 확인되지 않은 홍보 문구는 넣지 않습니다.
- 운영자 입력 문구는 공개 화면에 그대로 노출된다고 보고 작성합니다.

## 5. 광고 / 스폰서 / 제휴 표기 원칙

### 공통 원칙
- 사용자가 외부 이해관계를 알아차릴 수 있어야 합니다.
- 숨겨진 광고처럼 보이게 만들지 않습니다.
- 링크가 외부로 이동하면 화면 안에서 목적을 설명합니다.

### 표기 기준
- `Featured Listing`
  - 대회 자체를 더 눈에 띄게 보여주는 노출
  - 필요하면 설명 문구에 운영/협업 성격을 함께 적습니다.
- `스폰서`
  - 브랜드/파트너가 직접 연결된 정보형 슬롯
  - `광고 · 스폰서 링크` 같은 문구를 함께 둡니다.
- `제휴`
  - 외부 스토어/상품 카테고리 연결
  - `제휴 링크` 또는 `일부 링크는 제휴 링크로 연결될 수 있습니다.` 문구를 유지합니다.

### 피해야 하는 것
- 광고성 슬롯인데 일반 추천처럼만 보이게 쓰는 문구
- 실제로는 제휴 링크인데 표기를 빼는 경우
- 스폰서 슬롯인데 러너 정보처럼 위장하는 카피

## 6. 운영 응대 흐름

1. 문의 접수: `/advertise`
2. 운영 검토
   - 노출면 적합 여부
   - 문구 검수 가능 여부
   - 일정/링크/연락처 확인
3. 가격/계약 조건 별도 협의
4. 확정 후 `/ops/featured` 또는 관련 운영 화면에서 반영
5. 공개 화면에서 표기 상태 최종 점검

## 7. 아직 운영자가 확정해야 하는 항목

아래는 문서로 기준만 정리할 수 있고, 실제 값은 운영 결정이 필요합니다.

1. Featured listing 판매 가격
2. 판매 단위(기간, 슬롯별 차등 여부)
3. 문의 응답 SLA
4. 세금계산서 / 계약 처리 방식
5. 환불 또는 일정 변경 대응 정책

즉, 코드와 운영 가이드는 먼저 갖췄고, 실제 판매 조건은 별도 확정이 필요합니다.
