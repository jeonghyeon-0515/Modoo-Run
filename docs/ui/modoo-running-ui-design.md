# 모두의 러닝 UI 디자인 초안

- 관련 이슈: #15 모바일 우선 UI 디자인 시스템과 화면 구조를 설계한다

## 1. 디자인 컨셉
**러너가 대회 탐색부터 훈련 기록까지 빠르게 읽고, 쉽게 행동할 수 있는 맑고 단단한 정보 중심 인터페이스**

## 2. 디자인 원칙
1. 모바일 우선, 한 손 조작 우선
2. 정보는 빠르게 스캔되고 행동은 즉시 가능해야 함
3. 강조는 적게, 명확하게
4. 기능별 톤은 다르되 전체 시스템은 통일
5. 찾기 → 고르기 → 계획하기 → 실행하기 → 공유하기 흐름을 끊지 않기

## 3. 디자인 토큰
### 컬러 시스템
- Primary: Brand Blue — 메인 CTA, 활성 상태, 링크
- Secondary: Emerald — 완료/달성/긍정 상태
- Accent: Amber — 접수중, 마감임박, 주의 상태
- Neutral: Slate/Gray — 배경, 본문, 구분선, 메타 정보

### 타이포그래피
- 화면 제목: 24~32px
- 카드 제목: 16~18px
- 본문: 14~16px
- 메타 정보: 12~13px

### 간격 / radius / shadow
- 8px 그리드 시스템 사용
- 기본 간격: 8 / 12 / 16 / 24 / 32
- 카드와 패널은 중간 이상 radius 사용
- 그림자보다 border + 여백 중심

## 4. 공통 컴포넌트 규칙
- 카드: 상단 제목 / 중간 핵심 메타 / 하단 보조 정보 또는 CTA
- 배지: 접수중, 마감임박, 완료, 예정처럼 짧은 상태 전달용
- 필터 칩: 선택 여부가 즉시 보이도록 설계
- CTA: 화면당 강한 CTA 1개, 나머지는 보조 CTA
- 하단 네비게이션: 홈 / 대회 / 플랜 / 커뮤니티 중심

## 5. 화면별 UX 구조
### 홈
- 상단 헤더
- 히어로 요약 카드
- 주요 CTA 영역
- 추천 대회 카드 리스트
- 이번 달 계획 요약
- 커뮤니티 최신 글 미리보기
- 하단 네비게이션

### 대회 목록
- 제목 + 현재 조건 요약
- 검색/필터 바
- 정렬 옵션
- 대회 카드 리스트
- 더보기 또는 무한 스크롤

### 대회 상세
- 상단 요약 헤더
- 핵심 정보 카드
- 일정/접수 정보
- 상세 소개
- 원본 링크 / 수집 시각
- 관련 액션(계획 만들기 / 북마크 / 공유)

### 월간 플랜 캘린더
- 월 선택
- 이번 달 요약 카드
- 캘린더 뷰
- 날짜 선택 상세 패널
- 계획 관리 CTA

### 커뮤니티
- 카테고리 탭
- 글쓰기 CTA
- 글 목록
- 글 상세에서 본문 → 관련 대회 → 댓글 → 신고 순서

## 6. 마이크로카피
- 홈 히어로: 접수중인 대회부터 월간 훈련 계획까지, 러닝 준비를 한 곳에서 관리하세요.
- 대회 탐색: 지금 접수 가능한 마라톤 대회를 지역, 거리, 월별로 쉽게 찾아보세요.
- 플랜 캘린더: 목표 대회에 맞춰 훈련 계획을 세우고 날짜별로 실천 여부를 기록하세요.
- 달성 요약: 완료율과 연속 달성일을 확인하며 꾸준함을 이어가세요.
- 커뮤니티: 대회 준비, 훈련 기록, 후기와 팁을 다른 러너와 나눠보세요.

## 7. 상태별 처리
### 빈 상태
- 대회 없음: 아직 표시할 접수중 대회가 없어요.
- 플랜 없음: 이번 달 계획이 아직 없어요. 첫 계획을 만들어보세요.
- 커뮤니티 글 없음: 아직 글이 없어요. 첫 이야기를 시작해보세요.

### 로딩 상태
- 카드 스켈레톤 우선
- 화면 전체 블러 대신 섹션별 skeleton 사용

### 오류 상태
- 불러오기 실패: 데이터를 불러오지 못했어요. 잠시 후 다시 시도해 주세요.
- 저장 실패: 변경 내용을 저장하지 못했어요. 다시 시도해 주세요.
- 연결 오류: 네트워크 상태를 확인한 뒤 다시 시도해 주세요.

## 8. 구현 반영 메모
- 현재 `apps/web`의 홈 / 대회 / 플랜 / 커뮤니티 페이지에 위 구조를 반영했다.
- 대회 상세와 커뮤니티 상세도 모바일 우선 와이어프레임 수준으로 추가했다.
- 이후 #7, #8, #9, #10, #11 구현 시 이 문서를 기준으로 실제 데이터 연결과 상태 분기를 확장한다.
