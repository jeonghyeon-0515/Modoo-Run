# Modoo-Run 성장 / 수익화 / SEO 90일 전략

작성일: 2026-04-09
대상 서비스: 한국 러닝/마라톤 일정·상세·계획·커뮤니티 웹서비스

---

## 1. 목표

Modoo-Run을 단순 대회 목록 사이트가 아니라,
**러너가 실제로 참가할 대회를 고르고 준비하는 의사결정 도구**로 발전시킨다.

동시에 아래 3가지를 함께 달성한다.

1. **검색 유입 성장**
2. **재방문과 저장/계획 사용 증가**
3. **UX를 해치지 않는 수익화 시작**

---

## 2. 원칙

1. **러너 UX 우선**
   - 광고/제휴/스폰서는 탐색·비교·상세 확인 흐름을 방해하지 않는 범위에서만 운영한다.
2. **사람에게 도움이 되는 페이지를 만든다**
   - Google의 people-first / helpful content 원칙에 맞게, 단순 데이터 복사보다 실제 도움이 되는 설명과 판단 재료를 제공한다.
3. **검색 최적화는 기술 SEO + 고유 콘텐츠 + 내부 링크 구조로 간다**
   - 제목, 메타, structured data, sitemap, 포털 등록, 갱신 요청을 기본 체계로 만든다.
4. **수익화는 저리스크 순서로 시작한다**
   - 제휴 → 스폰서/featured listing → 디스플레이 광고 → 프리미엄 도구 순으로 간다.
5. **측정 가능한 것만 유지한다**
   - 검색 유입, 신청 클릭률, 저장률, 계획 생성률, 체류시간, 이탈률이 좋아지는 것만 유지한다.

---

## 3. 가장 중요한 의사결정 기준

1. **한국 러너가 반복 방문할 이유가 있는가**
2. **광고/수익화가 검색 품질과 브랜드 신뢰를 해치지 않는가**
3. **90일 안에 구현·측정·반복이 가능한가**

---

## 4. 추천 전략

### 추천안: SEO 중심 유틸리티 + 제휴 수익화 + 소규모 B2B

#### 1단계
- 검색 유입을 만들 수 있는 **대회 상세 고도화 + 편집형 랜딩 페이지**를 만든다.

#### 2단계
- 러너 준비물/장비/회복 관련 **제휴 수익화**를 시작한다.

#### 3단계
- 주최측 대상 **featured listing / 스폰서드 노출 / 직접 등록** 같은 B2B 실험을 한다.

#### 4단계
- 디스플레이 광고는 검색 유입이 늘어난 뒤, **정보형 페이지에만 저밀도**로 넣는다.

### 왜 이 안이 좋은가
- 현재 Modoo-Run은 이미 대회 상세/계획/커뮤니티가 있어 SEO 확장이 자연스럽다.
- 광고를 너무 빨리 붙이면 UX가 무너질 수 있다.
- 제휴와 featured listing은 상대적으로 초기 트래픽이 적어도 수익화 가능성이 있다.

## 5. 추가 기능 제안

### A. 가장 먼저 할 기능
1. **마감 임박 알림**
   - 7일 / 3일 / 1일 전 알림
2. **대회 비교 기능**
   - 일정, 거리, 지역, 접수기간, 난이도, 추천 대상 비교
3. **대회 상세 고유 정보 강화**
   - 교통, 주차, 물품보관, 급수, 컷오프, 준비물, 추천 대상
4. **개인화 추천**
   - 첫 10K / 하프 입문 / 여행 겸 참가 / 기록용 대회
5. **캘린더 내보내기**
   - Google / Apple / ICS

### B. 검색 유입과 재방문을 늘릴 기능
6. **지역별 landing page**
   - 서울 10km, 부산 하프, 봄 마라톤, 가을 마라톤
7. **거리별 landing page**
   - 5km / 10km / 하프 / 풀
8. **후기/평점**
   - 코스 난이도, 운영 만족도, 재참가 의향
9. **주간 추천 / 이번 달 추천 대회 큐레이션**
10. **주최측 직접 수정 요청 폼**

### C. 나중에 확장할 기능
11. **featured race**
12. **주최측 등록/운영 대시보드**
13. **러닝 크루 연계**
14. **프리미엄 러닝 플래너**

## 6. 수익화 전략

### 6-1. 1순위: 제휴 마케팅
한국 사용자 기준으로 현실적인 초기 수익화 방법.

추천 카테고리:
- 러닝화
- GPS 워치
- 러닝 양말 / 벨트 / 테이핑
- 보급젤 / 전해질 / 회복용품
- 원정 러너용 숙소 / 교통

운영 방식:
- 대회 목록/상세 핵심 화면에 과하게 붙이지 않는다.
- 대신 아래 같은 콘텐츠 페이지에 연결한다.
  - 첫 10km 준비물
  - 하프/풀 준비 체크리스트
  - 장거리 대회 필수 장비
  - 여름/겨울 대회 준비 팁

후보:
- Coupang Partners 같은 한국형 제휴 모델 우선 검토

### 6-2. 2순위: 스폰서 / featured listing
광고보다 UX 훼손이 적고, 적은 트래픽에서도 단가를 올릴 수 있다.

상품 예시:
- 메인 featured race
- 지역 허브 상단 노출
- 이번 주 추천 슬롯
- 뉴스레터 스폰서

주의:
- 반드시 `광고`, `스폰서`, `추천` 같은 표시는 명확히 해야 한다.

### 6-3. 3순위: 디스플레이 광고
Google Publisher Policies를 고려하면,
얇은 페이지·복제 페이지·광고가 콘텐츠를 가리는 구조는 피해야 한다.

권장 위치:
- 검색 유입이 큰 정보형 문서
- 준비 가이드
- 후기/리뷰 아카이브
- 지역/거리별 에버그린 랜딩

비권장 위치:
- 홈 첫 화면
- 대회 목록 상단
- 상세 hero 바로 아래
- CTA와 헷갈리는 위치

즉,
**의사결정 화면에는 광고를 최소화하고,
콘텐츠 소비 화면에 광고를 집중**한다.

### 6-4. 4순위: B2B
주최측 대상 상품:
- 직접 등록
- featured listing
- 참가자 유입 리포트
- 소개 페이지 제작 상품
- 마감 임박 알림/홍보 묶음 패키지

## 7. 홍보 전략

### 7-1. 검색 기반 홍보
- 지역/거리/시즌/마감 기준 랜딩 페이지 운영
- 상세 페이지마다 고유 설명 강화
- 내부 링크 허브 구성

### 7-2. 채널 홍보
1. **네이버 블로그 / 포스트**
   - 이번 주 접수 마감 대회
   - 봄 마라톤 추천
   - 서울/부산 추천 대회
2. **인스타 릴스 / 유튜브 쇼츠**
   - 이번 주말 대회 3개
   - 초보자용 첫 10km 추천
3. **러닝 크루/카페 제휴**
   - 지역 러닝 크루와 상호 링크/소개
4. **뉴스레터**
   - 마감 임박 / 이번 주 추천 / 시즌 추천
5. **주최측 상호 링크 확보**
   - 공식 페이지에서 Modoo-Run 상세 링크 유도

### 7-3. 포지셔닝 문구
- 단순히 “국내 대회 일정 모음”보다
- **“내가 실제로 뛸 대회를 고르는 가장 쉬운 방법”**
이 더 강하다.

## 8. 포털 상위노출 전략

## 8-1. Google
Google Search Central 기준 핵심:
- Search Console 연결
- sitemap 제출
- robots / canonical / meta 정리
- title / description 고유화
- internal link 구조 개선
- structured data 적용
- people-first content 강화

### 특히 중요한 것
대회 상세 페이지가 단순 복사본이면 약하다.
반드시 아래 같은 **고유 정보**를 추가해야 한다.
- 추천 대상 (입문/기록/가족/여행)
- 교통/주차/숙소 팁
- 준비물/주의사항
- 코스 특성 / 난이도
- 작년 후기 요약
- 주변 관광/맛집/동선

### 추가 권장
- `Event` structured data
- `Organization`, `WebSite` structured data
- Search Console에서 색인/클릭 확인
- 주요 페이지 recrawl 요청

## 8-2. Naver
Naver Search Advisor 기준 핵심:
- 사이트 등록
- 소유 확인
- 고유한 `<title>`
- 메타 설명 개선
- H1 구조 정리
- 이미지 alt
- 페이지 갱신 요청 활용

### 특히 중요한 것
- 문서 고유성
- 제목/설명문 품질
- 지속적 갱신
- 네이버 생태계 내 보조 채널 활용

권장:
- 본사이트 + 네이버 블로그/포스트 조합
- 블로그는 요약/큐레이션,
  본사이트는 상세/도구/저장/비교 흐름 담당

## 13. ADR

### Decision
SEO 중심 러닝 의사결정 서비스로 키우고,
수익화는 **제휴 → featured listing/B2B → 광고** 순으로 간다.

### Drivers
- 현재 제품 구조와 가장 잘 맞음
- UX 훼손이 적음
- 초기 트래픽이 작아도 시작 가능

### Alternatives considered
- 광고 우선 미디어형
- 구독형 먼저
- B2B 단독형

### Why chosen
- 광고 우선은 UX와 브랜드 신뢰 리스크가 큼
- 구독은 초기 가치 증명 전환이 어려움
- B2B 단독은 영업 부담이 큼
- 그래서 **SEO + 제휴 + 제한적 B2B 확장**이 가장 현실적

### Consequences
- 콘텐츠 품질 기준이 높아짐
- 상세 페이지 고도화가 중요해짐
- 단기 수익보다 검색/재방문 지표를 먼저 관리해야 함

### Follow-ups
- Search Console / Naver 운영 상태 점검
- structured data / sitemap / metadata 구현
- 비교 / 알림 / 캘린더 기능 우선순위 정리
- 제휴 문서/스폰서 상품안 작성
