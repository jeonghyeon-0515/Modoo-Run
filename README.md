# 모두의 러닝

> **대회 탐색부터 훈련 계획, 달성 체크, 러너 커뮤니티까지 연결하는 모바일 우선 러닝 웹앱**

모두의 러닝은 마라톤 대회 정보를 보기 쉽게 정리하고, 사용자가 목표 대회를 중심으로 월간 훈련 계획을 세우고 실천 과정을 기록하며, 다른 러너와 준비 과정과 후기를 나눌 수 있도록 설계된 서비스입니다.

---

## 왜 만들까

지금의 마라톤 정보 탐색 경험은 대회 일정 확인에 머무는 경우가 많습니다.

사용자는 보통 다음 흐름을 반복합니다.
1. 접수중인 대회를 찾는다.
2. 어떤 대회에 나갈지 고른다.
3. 대회 날짜에 맞춰 훈련 계획을 세운다.
4. 계획을 지켰는지 확인한다.
5. 준비 과정과 후기를 다른 러너와 공유한다.

하지만 이 흐름이 한 서비스 안에서 자연스럽게 이어지는 경우는 드뭅니다.

**모두의 러닝은 이 흐름을 하나로 연결하는 것**을 목표로 합니다.

---

## 우리가 제공하려는 경험

### 1. 접수중인 대회를 빠르게 찾는다
- 접수중인 마라톤 대회만 모아서 확인
- 지역, 월, 거리, 접수 상태 기준 필터링
- 모바일에서도 읽기 쉬운 카드형/리스트형 정보 구조

### 2. 대회 정보를 이해하기 쉽게 본다
- 대회명, 일정, 장소, 종목, 주최, 접수기간, 홈페이지, 기타 소개를 한 화면에서 정리
- 레거시 페이지보다 읽기 쉬운 상세 화면 제공
- 원본 링크와 마지막 수집 시각 함께 제공

### 3. 대회를 기준으로 훈련 계획을 세운다
- 월별 러닝 계획 캘린더 제공
- 날짜별 목표 거리와 훈련 내용을 기록
- 대회 일정과 개인 훈련 일정을 함께 관리

### 4. 계획을 실천하고 성취를 확인한다
- 완료 / 부분 완료 / 건너뜀 상태 체크
- 월간 완료율 확인
- 연속 달성 streak 확인
- 목표 대회 기준 준비 진행률 확인

### 5. 다른 러너와 연결된다
- 자유게시판
- 대회 준비 게시판
- 후기 게시판
- 댓글, 신고, 관리자 숨김 기반의 기본 커뮤니티 운영

## 핵심 기능

### 대회 탐색
- 접수중 대회 목록
- 지역/월/거리/접수상태 필터
- 관심 대회 북마크
- 대회 상세 재구성 페이지

### 훈련 계획
- 월별 플랜 캘린더
- 날짜별 훈련 계획 입력
- 계획 수정/삭제
- 대회와 훈련 일정 연결

### 달성 관리
- 계획 달성 체크
- 월간 완료율 요약
- 연속 달성 streak
- 준비 진행률 시각화

### 커뮤니티
- 게시글 작성/조회
- 댓글 작성
- 신고 기능
- 관리자 숨김 처리

## 모두의 러닝의 차별점
- **대회 검색에서 끝나지 않는다**: 대회 선택 후 계획 수립과 실행 관리까지 이어진다.
- **모바일 우선 경험을 전제로 설계한다**: 이동 중에도 대회 탐색과 체크가 쉬워야 한다.
- **개인 계획과 공개 대회 정보를 연결한다**: 관심 대회를 본 뒤 바로 나만의 월간 훈련 계획으로 넘어갈 수 있다.
- **정보 재구성에 집중한다**: 단순 링크 모음이 아니라 읽기 쉬운 형태로 다시 보여준다.

## 데이터 수집 방향
- 접수중 대회 정보는 Marathon Online 계열 페이지를 참고해 수집합니다.
- 실제 수집 대상은 `roadrun.co.kr`의 목록/상세 페이지입니다.
- 서비스 안에서는 원본 링크와 수집 시각을 함께 제공합니다.
- 데이터 품질과 운영 안정성을 위해 수집 로그, 파싱 상태, 중복 방지 기준을 관리합니다.

> 외부 데이터 사용 정책과 허용 범위는 구현 전에 별도 확인이 필요합니다.

## 이런 사용자에게 적합합니다
- 접수중인 마라톤 대회를 자주 찾는 러너
- 목표 대회를 정하고 체계적으로 훈련하고 싶은 사용자
- 월 단위로 계획을 세우고 꾸준히 달성 여부를 관리하고 싶은 사용자
- 준비 과정과 후기를 다른 러너와 나누고 싶은 사용자

## 제품 로드맵

### Phase 1
- 대회 탐색
- 대회 상세
- 월별 계획 캘린더
- 달성 체크
- 기본 커뮤니티

### Phase 2
- 북마크/알림 강화
- 기록 대시보드
- 운영 기능 보강
- Flutter 앱 확장 검토

## 기술 방향
- **Frontend**: Next.js App Router, TypeScript, 모바일 우선 반응형 UI, PWA
- **Backend**: Supabase Auth, Postgres
- **배포**: Vercel
- **스케줄 수집**: Vercel Cron → ingestion 경계 → Supabase upsert
- **확장 전략**: 웹앱 우선 출시 후 Flutter 앱 확장 가능 구조 유지
