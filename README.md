# 모두의 러닝

모두의 러닝은 **마라톤 대회 탐색**, **월별 러닝 계획 수립**, **달성 체크**, **커뮤니티**를 한 곳에서 제공하는 모바일 우선 러닝 웹앱입니다.

핵심 목표는 다음 두 가지입니다.
- 접수중인 마라톤 대회를 기존 사이트보다 **더 빠르고 보기 쉽게** 탐색할 수 있게 한다.
- 대회 참가 전후의 계획, 실천, 기록, 커뮤니티 활동까지 **하나의 흐름**으로 연결한다.

---

## 우리가 만들 서비스

### 1. 접수중인 마라톤 대회 탐색
- 접수중인 대회만 빠르게 모아보기
- 지역, 월, 거리, 접수 상태 기준 필터링
- 카드형/리스트형 보기 전환
- 모바일에서도 한눈에 읽히는 대회 정보 구조

### 2. 대회 상세 재구성 화면
- 대회명, 일정, 장소, 종목, 주최, 접수기간, 홈페이지, 기타 소개를 보기 좋게 정리
- 원본 링크와 마지막 수집 시각 제공
- 기존 레거시 페이지보다 읽기 쉬운 상세 화면 제공

### 3. 월별 러닝 계획 캘린더
- 월 단위로 러닝 계획 세우기
- 날짜별 목표 거리/훈련 내용 입력
- 대회 일정과 개인 훈련 일정을 함께 확인
- 월간 계획을 한 화면에서 관리

### 4. 달성 체크와 성취도 분석
- 완료 / 부분 완료 / 건너뜀 상태 체크
- 월간 완료율 확인
- 연속 달성 streak 확인
- 대회 D-day 기준 준비 진행률 확인

### 5. 커뮤니티
- 자유게시판
- 대회 준비 게시판
- 후기 게시판
- 댓글, 신고, 관리자 숨김 기능 기반의 기본 커뮤니티 운영

---

## 주요 차별점
- **접수중 대회 중심 탐색**: 필요한 정보만 빠르게 본다.
- **모바일 우선 UI**: 휴대폰에서 더 편하게 본다.
- **탐색 → 계획 → 달성 → 공유** 흐름 연결: 대회 검색에서 끝나지 않는다.
- **대회 정보와 개인 계획 결합**: 관심 대회를 보고 바로 훈련 계획으로 이어갈 수 있다.

---

## MVP에서 먼저 제공할 기능
- 접수중 마라톤 대회 목록
- 대회 상세 페이지
- 관심 대회 북마크
- 월별 플랜 캘린더
- 계획 달성 체크
- 완료율 / streak 요약
- 기본 커뮤니티

---

## 이후 확장할 기능
- 접수 마감 알림
- 거리별 추천 훈련 템플릿
- 개인 기록 대시보드
- 대회별 후기/코스 팁 모음
- 크루/모임 기능
- 관리자용 대회 데이터 보정 도구
- Flutter 기반 모바일 앱 확장

---

## 데이터 수집 방식
- 접수중 대회 정보는 Marathon Online 계열 페이지를 참고해 수집한다.
- 실제 수집 대상은 `roadrun.co.kr`의 목록/상세 페이지다.
- 원본 링크와 수집 시각을 함께 제공한다.
- 데이터 품질과 운영 안정성을 위해 수집 로그, 파싱 상태, 중복 방지 기준을 관리한다.

> 외부 데이터 사용 정책과 허용 범위는 구현 전 별도 확인이 필요합니다.

---

## 기술 방향
- **Frontend**: Next.js App Router, TypeScript, 모바일 우선 반응형 UI, PWA
- **Backend**: Supabase Auth, Postgres
- **배포**: Vercel
- **스케줄 수집**: Vercel Cron → ingestion 경계 → Supabase upsert
- **확장 전략**: 웹앱 우선 출시 후 Flutter 앱 확장 가능 구조 유지

---

## 대상 사용자
- 마라톤 대회 정보를 자주 찾는 러너
- 목표 대회를 정하고 훈련 계획을 세우는 사용자
- 월간 단위로 꾸준히 러닝 습관을 관리하고 싶은 사용자
- 대회 후기와 준비 과정을 다른 러너와 공유하고 싶은 사용자
